面向 OpenCode 的多模型智能体编排插件。将单一 AI 转变为一支协调的开发团队。

核心理念：不拘泥于单个模型或单一供应商。用最适合的大脑做最合适的工作——Claude 做编排，GPT 做深度推理，Gemini 做前端……自动协同，无需手动切换。

哲学理念

人在回路 = 瓶颈

就像自动驾驶需要人类接管方向盘是系统故障一样——如果你需要不断修正 AI 的半成品代码、逐步指导它完成任务、反复澄清需求，那不是"人机协作"，那是 AI 未能履行其职责。OmO 的目标是让 agent 在无需人类干预的情况下完成工作。

代码不可区分

Agent 编写的代码应该与高级工程师编写的代码无法区分。不是"AI 生成的代码需要清理"，不是"不错的起点"——而是真正的、最终的、可投入生产的代码。遵循已有模式、合理处理错误、不画蛇添足。

最小化认知负担

人类只需要说他们想要什么。其余全是 agent 的工作。

两种模式实现这一点：Prometheus 模式（访谈式规划）和 Ultrawork 模式（只管做）。人类提供意图，系统提供结构与执行。

可预测、持续、可委派

理想的 agent 应该像编译器一样工作：Markdown 文档进去，工作代码出来。会话崩溃？用 /start-work 恢复。多天项目？上下文被保留。像委派任务给能干队友一样委派给 agent。

Agents 智能体系统

项目提供 11 个专业 AI Agent。每个都有独特的专长、优化的模型和工具权限。

核心 Agent (固定排序: Sisyphus → Hephaestus → Prometheus → Atlas)

Agent推荐模型角色
Sisyphus Claude Opus 4.7 / Kimi K2.6 / GLM 5.1 主编排器。规划、委派给专家 agent、激进并行执行。Todo 驱动工作流，扩展思考 (32k budget)。不半途而废。
Hephaestus GPT-5.5 自主深度工作者。给他目标而非配方。探索代码库、研究模式、端到端执行。灵感来自 AmpCode 的 deep 模式。
Oracle GPT-5.5 / Gemini 3.1 Pro / Claude Opus 4.7 只读高智商顾问。架构决策、代码审查、复杂调试。只读。不写不编不委派。
Librarian GPT-5.4 Mini / Qwen / MiniMax 多仓库分析。文档查找、开源实现示例。深度代码库理解与基于证据的回答。只读。
Explore GPT-5.4 Mini / Qwen / MiniMax 快速代码库探索。上下文 grep。速度优先。只读。
Multimodal Looker GPT-5.5 / Kimi K2.6 / GLM 4.6v 视觉内容专家。分析 PDF、图片、图表。仅限 read 工具。

分层架构

38 个 monorepo 包按严格层级组织。依赖规则：DAG 只能向下流动。

Web — 营销网站
packages/web/ — Next.js 15 + Cloudflare Workers
Platform — 平台二进制
11 平台二进制 (darwin/linux/windows × arch × variant)
Adapters — 适配器
omo-opencode/ + omo-codex/ — 薄包装导入 Core
Skills — 技能
shared-skills/ — SKILL.md 静态声明，无代码
MCP — 工具服务器
lsp-tools-mcp ast-grep-mcp 等 — stdio 进程边界
Core — 纯逻辑核心 (19 包)
无框架依赖，可隔离测试
