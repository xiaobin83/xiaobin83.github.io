1 用户视角 — Vikingbot 是什么？

最简单直接的理解

Vikingbot 是一个"能在任何聊天平台里回复你的 AI 助手"。

你可以在 Telegram、飞书、Discord、Slack、钉钉、QQ、WhatsApp、邮件等 10+ 个平台上和它对话。它不仅仅是聊天——它能读文件、写代码、搜网页、执行命令、生成图片、定时提醒你，还能自动学习并记住你的偏好。

2 两种运行模式

Gateway 模式 vs CLI 模式 — 同一套核心，不同的外壳

🌐 Gateway 模式 长期运行

ov gateway

长期运行的服务，启动 5 个并发任务：

  • Cron — 定时任务
  • Heartbeat — 心跳检测
  • Channels — 所有聊天平台
  • Agent Loop — 核心引擎
  • FastAPI Server — HTTP API

💬 CLI 模式 交互式

ov chat -m "hello"   # 单次
ov chat               # 交互式

核心流程与 Gateway 相同，只是用 ChatChannel 或 SingleTurnChannel 替代了外部平台。

💡 关键点：两种模式下，Agent Loop、Provider、Tool Registry、Session Manager 等核心组件完全复用。唯一的区别是 Channel 的不同。

📬 MessageBus — 解耦核心

两个独立的 asyncio.Queue：

  • inbound：Channel → Agent Loop 的消息
  • outbound：Agent Loop → Channel 的回复

Channels 和 Agent Loop 完全解耦，互不感知，只通过总线通信。

📡 Channel 的职责

  1. 连接聊天平台（WebSocket / Polling / Webhook）
  2. 接收用户消息 → 转换为 InboundMessage
  3. 发布到 MessageBus.inbound
  4. 接收 OutboundMessage 并发送回平台

4 Agent Loop — 核心思考循环

OpenClaw-like Agent 的本质 — ReAct (Reasoning + Acting) 循环

收到 InboundMessage │ ├─ 特殊命令处理 (/new, /compact, /remember, /help) │ ├─ 检查是否需要压缩记忆 (session > memory_window) │ ├─ 构建 Context (ContextBuilder.build_messages) │ ├── System Prompt (SOUL + TOOLS + 技能) │ ├── 用户记忆 (OpenViking 语义检索) │ ├── 用户画像 (USER.md) │ └── 对话历史 (最近 N 轮) │ ├─ 进入核心循环 _run_agent_loop() │ └─ 保存 Session → 发布 OutboundMessage

🔄 ReAct 循环实现

while iteration < max_iterations:
    iteration += 1

    # Step 1: 调用 LLM（带上工具定义）
    response = await provider.chat(
        messages=messages,
        tools=tools.get_definitions(),
    )

    if response.has_tool_calls:
        # Step 2-4: 执行工具，将结果放回消息列表
        results = await asyncio.gather(*[
            execute_single_tool(tc) for tc in response.tool_calls
        ])
        messages = context.add_tool_result(messages, ...)
        messages.append({"role": "user",
            "content": "Reflect on the results..."})
    else:
        # LLM 认为任务完成
        final_content = response.content
        break

工具执行流程

每个工具执行时自动接入追踪和钩子：

async def execute(name, params, session_key, ...):
    # 1. 查找工具
    tool = self._tools.get(name)

    # 2. Langfuse 追踪开始
    tool_span = langfuse.tool_call(name=name, input=params)

    # 3. 参数校验
    errors = tool.validate_params(params)

    # 4. 执行
    result = await tool.execute(tool_context, **params)

    # 5. 执行后 Hook
    hook_result = await hook_manager.execute_hooks(
        context=HookContext(event_type="tool.post_call"),
        tool_name=name, params=params, result=result,
    )

7 LLM Provider — 模型提供者

统一的 LLM 调用抽象层

📐 抽象接口

class LLMProvider:
    async def chat(
        messages: list[dict],
        tools: list[dict] | None,
        model: str | None,
    ) -> LLMResponse:
        pass

# LLMResponse 包含:
# - content: 文本回复
# - tool_calls: 工具调用请求列表
# - reasoning_content: 思考过程 (Kimi/DeepSeek)

⚡ 关键设计：记忆窗口 + 自动压缩 — 会话超过 50 轮时，自动触发记忆压缩（调用 LLM 提取摘要），将旧消息归档到 OpenViking 中长期记忆。

9 Subagent — 子 Agent

后台任务执行的"Agent 层级协作"机制

🧠 主 Agent
调用 spawn 工具 → 创建子 Agent
⬇
⚙️ 子 Agent
独立运行（最多 15 轮）→ 结果通知主 Agent

🔄 独立循环

子 Agent 拥有独立的 ToolRegistry，不包含 message 和 spawn 工具。

🎯 专注任务

限制最多 15 轮迭代，专注于完成单一任务。

📬 结果通报

完成后通过 MessageBus 将结果注入给主 Agent，由主 Agent 自然地呈现给用户。

10 Cron & Heartbeat — 定时服务

让 Agent 具备自主行动能力

⏰ Cron Service

支持三种调度方式：

  • every — 固定间隔（如每 2 小时）
  • cron — cron 表达式（如每天 9:00）
  • at — 一次性定时（ISO 格式时间）

Agent 通过 cron 工具自主创建和管理定时任务。

❤️ Heartbeat Service

定期检查 HEARTBEAT.md 文件：

  • 读取待办任务清单
  • 构造 prompt 触发 Agent 执行
  • 结果可自动发送到指定频道

两者都通过 AgentLoop.process_direct() 触发。

11 完整数据流

从用户消息到 AI 回复的完整链路

用户发消息 "帮我查一下今天上海天气" │ ▼ 📡 Channel (如 FeishuChannel) │ 1. 创建 InboundMessage │ 2. bus.publish_inbound(msg) ▼ 🔗 MessageBus.inbound 队列 │ ▼ 🧠 AgentLoop.run() → _process_message(msg) │ ├─ 1. 获取 / 创建 Session ├─ 2. ContextBuilder.build_messages() │ ├─ 构建 System Prompt (SOUL + TOOLS + 技能 + 用户画像) │ └─ 注入 OpenViking 记忆 ├─ 3. _run_agent_loop() ← ReAct 循环 │ ├─ LLM.chat() → "需要调用 web_search" │ ├─ ToolRegistry.execute("web_search", {"query": "上海天气"}) │ ├─ LLM.chat() → "上海今天晴，25°C" │ └─ 返回 final_content ├─ 4. Session.add_message() → 保存对话 ├─ 5. bus.publish_outbound(OutboundMessage) │ ▼ 🔗 MessageBus.outbound 队列 │ ▼ 📡 ChannelManager._dispatch_outbound() │ 找到对应 Channel ▼ 📡 Channel.send() │ ▼ 用户收到回复 "上海今天晴，25°C ☀️"

🎯 核心设计理念：通过 MessageBus 解耦 Channels 和 Agent Loop，通过 ReAct 循环让 LLM 自主决定何时调用工具，通过 Context Builder 将记忆、技能、用户画像统一注入到每一次对话中。
