pi-coding-agent 设计文档

npm 包: @earendil-works/pi-coding-agent | 源码目录: packages/coding-agent/

概述

@earendil-works/pi-coding-agent 是 Pi 的终端编码代理 CLI。它通过命令行参数解析、会话管理、扩展系统、工具注册和多种运行模式，将 @earendil-works/pi-agent-core(代理运行时)和 @earendil-works/pi-ai(LLM API 层)编排为一个完整的终端应用。

核心设计原则:

  • SDK 作为单一入口: CLI(main.ts)不直接操作代理，而是通过 createAgentSession()(packages/coding-agent/src/core/sdk.ts:166)构建服务，再由 createAgentSessionRuntime()(packages/coding-agent/src/core/agent-session-runtime.ts)封装为运行时。
  • AgentSession 共享于所有模式: 交互式、print、JSON、RPC 四种模式共享同一个 AgentSession 实例(packages/coding-agent/src/core/agent-session.ts)，模式差异仅在于 I/O 层。
  • 扩展驱动可定制性: 内置工具、命令、UI 组件均通过 ExtensionAPI(packages/coding-agent/src/core/extensions/types.ts)暴露，用户可以替换或扩展任意部分。

CLI 入口流程

启动路径: main.ts -> parseArgs -> resolveAppMode -> 模式分发

CLI 的 main() 函数(packages/coding-agent/src/main.ts:468)执行以下流程:

  1. 离线和 Windows 自更新清理(第470-478行): 处理 --offline / PI_OFFLINE 环境变量，Windows 平台清理隔离目录。

  2. bootstrap SettingsManager(第482行): 以 projectTrusted: false 创建初始设置管理器，仅用于读取 httpProxy 等全局配置，不加载项目级资源。

  3. 包命令分发(第486行): handlePackageCommand() 处理 pi install/remove/update/list 等子命令，命中则直接退出。

  4. config 命令分发(第499行): handleConfigCommand() 打开 TUI 配置界面。

  5. 参数解析(packages/coding-agent/src/cli/args.ts:63, 第503行): parseArgs() 将 process.argv 解析为 Args 结构体。未知的 --flag 存入 unknownFlags: Map<string, boolean | string>，供扩展注册的 CLI flag 消费。

  6. 模式解析(main.ts:100-111 的 resolveAppMode()):

  7. --mode rpc -> rpc 模式
  8. --mode json -> json 模式
  9. -p 或非 TTY stdin/stdout -> print 模式

  10. 否则 -> interactive 模式

  11. 迁移和首次设置(第549-560行): runMigrations() 处理认证迁移; showFirstTimeSetup() 在首次启动时让用户选择主题和分析同意。

  12. SessionManager 创建(main.ts:264-350 的 createSessionManager()): 根据 CLI 参数选择会话来源--新建、--session 指定、--continue 继续、--resume 选择、--fork 分支、--no-session 内存模式。--session 和 --fork 支持 UUID 前缀匹配，先在当前项目查找，再全局搜索。

  13. 项目信任(第596-658行): 创建 ProjectTrustStore。createAgentSessionRuntime 的工厂函数中，resourceLoaderReloadOptions.resolveProjectTrust 在项目未信任时延迟解析--此时仅加载全局扩展和 CLI -e 扩展，项目级扩展在信任后再加载。

  14. 构建 Runtime(第736行): createAgentSessionRuntime(createRuntime, { cwd, agentDir, sessionManager }) 执行工厂函数，创建完整的 AgentSessionRuntime。

  15. 模式分发(第806-853行):

    • rpc -> runRpcMode(runtime)(packages/coding-agent/src/modes/rpc/rpc-mode.ts)
    • interactive -> new InteractiveMode(runtime, {...}).run()(packages/coding-agent/src/modes/interactive/interactive-mode.ts)
    • 其他 -> runPrintMode(runtime, {...})(packages/coding-agent/src/modes/print-mode.ts)

参数类型定义

Args 接口(packages/coding-agent/src/cli/args.ts:12-55)定义了所有 CLI 参数，关键字段:

字段 类型 说明
provider string? 提供商名称
model string? 模型 pattern，支持 provider/id:thinking 格式
thinking ThinkingLevel? 思考等级: off/minimal/low/medium/high/xhigh
mode Mode? 输出模式: text/json/rpc
tools string[]? 工具白名单
excludeTools string[]? 工具黑名单
noTools boolean? 禁用所有工具
noBuiltinTools boolean? 仅禁用内置工具
extensions string[]? 额外扩展路径(-e)
session string? 指定会话路径或 UUID
fork string? 分支会话路径或 UUID
sessionId string? 精确会话 ID
messages string[] 位置消息参数
fileArgs string[] @ 前缀的文件参数
unknownFlags Map<string, boolean|string> 未知 flag，供扩展消费

core/ 子系统

packages/coding-agent/src/core/ 目录包含所有模式共享的核心子系统。

AgentSession

源文件: packages/coding-agent/src/core/agent-session.ts

AgentSession 是所有运行模式共享的代理生命周期抽象(文件头部注释说明: shared between all run modes (interactive, print, rpc))。它封装:

  • 代理状态访问: 通过 agent getter 暴露底层 Agent 实例
  • 事件订阅与自动会话持久化: 订阅 Agent 事件并写入 SessionManager
  • 模型和思考等级管理: setModel()、setThinkingLevel()，自动 clamp 到模型支持的等级
  • Compaction: 手动(compact())和自动(autoCompact())上下文压缩
  • Bash 执行: 通过 executeBashWithOperations()(packages/coding-agent/src/core/bash-executor.ts)执行命令
  • 会话切换和分支: 支持 /tree 导航、分支切换

AgentSessionRuntime

源文件: packages/coding-agent/src/core/agent-session-runtime.ts

AgentSessionRuntime 是 AgentSession + 服务的容器，提供模式无关的运行时:

  • session: AgentSession -- 代理会话实例
  • services: AgentSessionServices -- 包含 settingsManager、modelRegistry、authStorage、resourceLoader
  • extensionsResult: LoadExtensionsResult -- 扩展加载结果
  • diagnostics: AgentSessionRuntimeDiagnostic[] -- 启动诊断信息
  • dispose() -- 清理扩展和会话资源

sdk.ts

源文件: packages/coding-agent/src/core/sdk.ts

SDK 入口，导出 createAgentSession() 和 CreateAgentSessionOptions。它是外部嵌入 Pi 的主要 API。

CreateAgentSessionOptions(第34-83行)定义了所有可配置项: cwd、agentDir、authStorage、modelRegistry、model、thinkingLevel、scopedModels、noTools、tools、excludeTools、customTools、resourceLoader、sessionManager、settingsManager。SDK 使用详情参见 SDK 文档。

system-prompt.ts

源文件: packages/coding-agent/src/core/system-prompt.ts

构建 agent 的 system prompt。组装流程: 加载内置模板 -> 合并 SYSTEM.md -> 拼接上下文文件 -> 追加 skills prompt -> 追加扩展片段。

model-registry.ts

源文件: packages/coding-agent/src/core/model-registry.ts

ModelRegistry 管理所有可用模型。它包装 @earendil-works/pi-ai 的 Models 集合，额外处理: 从 models.json 加载自定义模型、按 provider 分组查询、模糊搜索(find() 支持 pattern 匹配)、与 AuthStorage 集成自动解析 API key / OAuth token。

session-manager.ts

源文件: packages/coding-agent/src/core/session-manager.ts

SessionManager 管理会话文件的持久化。详见下方--会话管理--章节。

settings-manager.ts

源文件: packages/coding-agent/src/core/settings-manager.ts

SettingsManager 管理全局(~/.pi/agent/settings.json)和项目级(.pi/settings.json)设置。项目级设置覆盖全局设置。详见 设置文档。

resource-loader.ts

源文件: packages/coding-agent/src/core/resource-loader.ts

ResourceLoader 负责发现和加载所有可定制资源:

  • 扩展(extensions): ~/.pi/agent/extensions/、.pi/extensions/、pi packages
  • 技能(skills): ~/.pi/agent/skills/、~/.agents/skills/、.pi/skills/、.agents/skills/、pi packages
  • 提示模板(prompt templates): ~/.pi/agent/prompts/、.pi/prompts/、pi packages
  • 主题(themes): ~/.pi/agent/themes/、.pi/themes/、pi packages
  • 上下文文件(context files): AGENTS.md / CLAUDE.md(从 cwd 向上遍历)

skills.ts

源文件: packages/coding-agent/src/core/skills.ts

Skill 系统实现。Skill 遵循 Agent Skills 标准，是以 SKILL.md 文件定义的按需能力包。加载后，skill 的 prompt 内容被注入 system prompt。详见 技能文档。

prompt-templates.ts

源文件: packages/coding-agent/src/core/prompt-templates.ts

提示模板系统。模板是 Markdown 文件，支持 {{variable}} 占位符。用户在编辑器中输入 /name 即可展开对应模板。

slash-commands.ts

源文件: packages/coding-agent/src/core/slash-commands.ts

斜杠命令注册表。内置命令(/login、/model、/settings、/tree 等)和扩展注册的命令均通过此模块管理。

package-manager.ts

源文件: packages/coding-agent/src/core/package-manager.ts

Pi 包管理器。支持从 npm、git、本地路径安装 pi package。安装位置: 全局(~/.pi/agent/npm/、~/.pi/agent/git/)或项目本地(.pi/npm/、.pi/git/)。

trust-manager.ts

源文件: packages/coding-agent/src/core/trust-manager.ts

项目信任管理。ProjectTrustStore 读/写 ~/.pi/agent/trust.json，存储用户对项目目录的信任决策。

event-bus.ts

源文件: packages/coding-agent/src/core/event-bus.ts

EventBus 是扩展系统的事件中枢。它提供类型安全的事件发布/订阅机制，所有扩展生命周期事件(session_start、message_start、tool_call、turn_end 等)通过它分发。

compaction/

源文件: packages/coding-agent/src/core/compaction/

上下文压缩子系统。当会话历史接近模型上下文窗口限制时触发: shouldCompact() 判断是否需要压缩，prepareCompaction() 准备消息条目，compact() 调用模型生成摘要，generateBranchSummary() 为非活跃分支生成摘要。压缩是有损的--完整历史保留在 JSONL 文件中，可通过 /tree 回溯。详见 压缩文档。

四种运行模式

Pi 支持四种运行模式，共享同一个 AgentSession 实例，差异仅在于 I/O 层和用户交互方式。

模式总览

模式 CLI 触发 入口 I/O 方式 适用场景
interactive pi(默认) InteractiveMode(modes/interactive/interactive-mode.ts) TUI(终端 UI) 日常开发交互
print pi -p / 管道输入 runPrintMode()(modes/print-mode.ts) stdout 文本输出 脚本集成、单次问答
json pi --mode json runPrintMode()(mode=json) stdout JSON 事件流 程序化消费事件
rpc pi --mode rpc runRpcMode()(modes/rpc/rpc-mode.ts) stdin/stdout JSONL 协议 进程嵌入(非 Node.js)

Interactive 模式

交互模式通过 InteractiveMode 类(packages/coding-agent/src/modes/interactive/interactive-mode.ts)实现。它构建在 @earendil-works/pi-tui 之上，提供完整的终端 UI:

  • 启动头部: 显示快捷键提示、加载的 AGENTS.md、prompt templates、skills、extensions
  • 消息区域: 渲染用户消息、assistant 响应、工具调用/结果、通知、错误、扩展 UI
  • 编辑器: 用户输入区域，支持 @ 文件引用、路径补全、多行编辑(Shift+Enter)、外部编辑器(Ctrl+G)、图片粘贴(Ctrl+V)、bash 命令(!command / !!command)
  • 页脚: 显示工作目录、会话名、token/缓存用量、成本、上下文使用率、当前模型

数据流:

用户输入 -> Editor -> InteractiveMode.prompt()
  -> AgentSession.prompt() -> Agent.runAgentLoop()
  -> LLM API (通过 @earendil-works/pi-ai)
  -> 事件流 -> InteractiveMode 渲染到 TUI
  -> 工具调用 -> 执行 -> 结果送回 Agent
  -> 最终响应 -> 渲染到消息区域

扩展可替换编辑器、添加 widget、状态行、自定义页脚或覆盖层。还支持消息队列(steering/follow-up)、/tree 树导航、项目信任提示、中途引导。

Print 模式

Print 模式(packages/coding-agent/src/modes/print-mode.ts)用于单次问答和脚本集成:

  • -p / --print: 发送 prompt，输出最终响应文本，退出
  • --mode json: 发送 prompt，输出所有事件为 JSON 行(每行一个 JSON 对象)
  • 支持管道输入: cat file | pi -p summarize 将 stdin 内容合并到初始 prompt
  • takeOverStdout()(packages/coding-agent/src/core/output-guard.ts)接管 stdout

数据流差异:

  • 不渲染 TUI，直接输出到 stdout
  • 无用户交互--不支持中途引导(steering)和跟进(follow-up)
  • 不显示项目信任提示，使用 defaultProjectTrust 设置
  • runPrintMode() 返回退出码(非零表示工具执行失败)

JSON 模式

JSON 模式是 Print 模式的变体(mode=json)。runPrintMode() 输出所有 AgentSessionEvent 为 JSON 行(每行一个 JSON 对象)，包括 message_start、text_delta、tool_call、tool_result、message_end 等事件。适合程序化消费 agent 事件流。

RPC 模式

RPC 模式(packages/coding-agent/src/modes/rpc/rpc-mode.ts)通过 stdin/stdout 的 LF-delimited JSONL 协议提供进程集成能力。协议定义(packages/coding-agent/src/modes/rpc/rpc-types.ts):

  • Commands: 客户端发送的 JSON 对象，包含 type 字段(prompt、steer、follow_up、set_model 等)和可选的 id 用于关联
  • Responses: 服务器返回的 JSON 对象，包含 type: response、command、success、data/error
  • Events: AgentSessionEvent 对象流式发送
  • Extension UI: 扩展 UI 请求通过 extension_ui_request / extension_ui_response 交互

RPC 模式使用严格的 LF 分隔，客户端不能使用 Node readline(它会按 Unicode 行分隔符拆分 JSON 内容)。详见 RPC 文档。

扩展系统架构

ExtensionAPI

ExtensionAPI(packages/coding-agent/src/core/extensions/types.ts)是扩展的入口接口，提供以下能力域:

  • 工具注册: registerTool() 注册 LLM 可调用的工具，replaceBuiltinTool() 替换内置工具
  • 命令注册: registerCommand() 注册斜杠命令
  • 快捷键注册: registerShortcut() 注册全局快捷键
  • 事件订阅: on() / off() 订阅生命周期事件(session_start、message_start、tool_call、turn_end、session_shutdown 等)
  • UI 组件: addWidget()、addFooterSection()、showDialog()、addOverlay()、replaceEditor()
  • Provider 注册: registerProvider() 注册自定义 LLM provider
  • System Prompt: appendSystemPrompt() 追加 system prompt 片段
  • CLI Flags: registerFlag() 注册扩展 CLI flag
  • Context: 通过 ExtensionContext 访问 cwd、sessionManager、modelRegistry、当前模型、isIdle、shutdown 等

ExtensionFactory 和 ExtensionRunner

扩展文件导出一个默认函数(可 async):

export default function (pi: ExtensionAPI) {
  pi.registerTool({ name: "deploy", ... });
  pi.registerCommand("stats", { ... });
  pi.on("tool_call", async (event, ctx) => { ... });
}

ExtensionRunner(packages/coding-agent/src/core/extensions/)负责加载扩展模块、管理事件订阅、创建扩展上下文。加载顺序:

  1. CLI -e 扩展(总是加载)
  2. 全局扩展(~/.pi/agent/extensions/)
  3. 项目扩展(.pi/extensions/)--仅项目信任后加载
  4. Pi package 扩展--仅项目信任后加载

详见 扩展系统文档。

内置工具

Pi 默认提供 7 个内置工具(通过 packages/coding-agent/src/core/tools/):

工具名 源文件 功能 默认启用
read tools/read.ts 读取文件内容 是
write tools/write.ts 创建/覆写文件 是
edit tools/edit.ts 精确字符串替换编辑 是
bash tools/bash.ts 执行 bash 命令 是
grep tools/grep.ts 正则搜索文件内容 否
find tools/find.ts glob 模式查找文件 否
ls tools/ls.ts 列出目录内容 否

工具控制 CLI 选项:

  • --no-tools / -nt: 禁用所有工具(内置 + 扩展)
  • --no-builtin-tools / -nbt: 仅禁用内置工具，保留扩展工具
  • --tools read,grep / -t read,grep: 白名单模式，仅启用指定工具
  • --exclude-tools bash / -xt bash: 黑名单模式，禁用指定工具

会话管理

JSONL 格式

会话以 JSONL 文件存储于 ~/.pi/agent/sessions/<encoded-cwd>/。每行一个 JSON 对象，通过 id 和 parentId 字段形成树状分支结构(packages/coding-agent/src/core/session-manager.ts):

  • session 条目: 会话头部，包含 version、id、timestamp、cwd
  • message 条目: 用户消息或 assistant 消息，包含 parentId 指向前一条目
  • branch_summary 条目: 分支摘要(compaction 产物)
  • custom 条目: 扩展写入的自定义数据

SessionManager API

SessionManager(packages/coding-agent/src/core/session-manager.ts)提供:

  • SessionManager.create(cwd) -- 新建会话
  • SessionManager.open(path) -- 打开已有会话
  • SessionManager.continueRecent(cwd) -- 继续最近会话
  • SessionManager.inMemory(cwd) -- 内存模式(不持久化)
  • SessionManager.forkFrom(sourcePath, targetCwd) -- 分支会话到新项目
  • SessionManager.list(cwd) -- 列出当前项目会话
  • SessionManager.listAll() -- 全局搜索所有会话

树状分支

每个会话条目有 id 和 parentId，支持原地分支:

  • /tree 命令在交互模式中浏览和跳转到任意历史点
  • /fork 命令从历史用户消息创建新会话文件
  • /clone 命令复制当前活跃分支到新会话文件
  • --fork <path|id> CLI 选项从命令行分支会话

详见 会话格式文档。

安全与容器化

Pi 默认以启动用户的权限运行，不内置文件系统、进程、网络或凭证访问限制。如需更强隔离:

  • Gondolin 扩展: 将 pi 和 provider 认证保留在宿主机，将内置工具和 ! 命令路由到本地 Linux micro-VM
  • Plain Docker: 在本地容器中运行完整 pi 进程
  • OpenShell: 在策略控制的沙箱中运行完整 pi 进程

项目信任机制(trust-manager.ts)控制是否加载项目级扩展和设置。详见 安全文档 和 容器化文档。
