Pi 文档索引

项目简介

Pi 是一个极简终端编程助手（terminal coding harness），核心保持最小化，通过 TypeScript 扩展（extension）、技能（skill）、提示模板（prompt template）和主题（theme）机制实现高度可定制。Pi 的设计哲学是"适应你的工作流，而不是反过来"——它不内置子代理、计划模式、权限弹窗等功能，而是提供强大的扩展 API，让你按需构建或从社区安装。

Pi 运行于四种模式：交互式终端、单次输出（-p）、JSON 事件流（--mode json）和 RPC 模式（--mode rpc），并可通过 SDK 嵌入 Node.js 应用。Pi 是一个 monorepo，包含五个 npm 包，从底层多提供商 LLM API 到上层编码代理 CLI，逐层构建。

中文术语表

Pi 文档以英文编写为主，以下术语在中文技术讨论中的推荐翻译：

英文 中文推荐 说明
agent 代理 执行工具调用和对话循环的有状态运行时
coding agent 编码代理 面向软件开发场景的代理，Pi 的核心场景
compaction 上下文压缩 对历史消息进行摘要以节省上下文窗口
extension 扩展 TypeScript 模块，可注册自定义工具、命令、事件处理器和 UI 组件
harness 运行时框架 Pi 作为"代理运行时"的整体定位
hook 钩子 扩展生命周期中可注入逻辑的挂载点
model 模型 LLM 模型，包含提供商、API 类型、上下文窗口等元数据
provider 提供商 LLM 服务提供方（Anthropic、OpenAI、Google 等），负责认证和模型目录
session 会话 一次对话的完整记录，以 JSONL 文件存储，支持分支树结构
skill 技能 遵循 Agent Skills 标准的按需能力包，通过 /skill:name 或自动加载
steering 中途引导 代理执行工具期间用户插入的纠正/引导消息
stream 流 模型响应的增量式传输，通过事件（text_delta、toolcall_delta 等）逐步推送
tool 工具 代理可调用的外部能力（读文件、执行命令等）
turn 轮次 一次 LLM 调用及其后续工具执行构成的完整交互周期
