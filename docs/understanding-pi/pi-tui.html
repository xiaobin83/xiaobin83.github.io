pi-tui 终端 UI 框架设计

@earendil-works/pi-tui（源码 packages/tui/）是 Pi monorepo 的终端 UI 框架层，提供差分渲染（differential rendering）、组件系统和键盘输入处理。它是整个 Pi 技术栈的最底层包，无 workspace 依赖，被 pi-coding-agent 直接依赖。

TUI 主类与差分渲染

TUI（packages/tui/src/tui.ts）是整个框架的核心，继承自 Container，管理组件树、渲染循环、焦点和 overlay 系统。

Component 接口

所有 UI 组件实现 Component 接口（packages/tui/src/tui.ts）：

export interface Component {
  render(width: number): string[];  // 每行宽度必须 <= width
  handleInput?(data: string): void; // 键盘输入（仅获得焦点时调用）
  wantsKeyRelease?: boolean;         // 是否接收按键释放事件（Kitty 协议 flag 2）
  invalidate(): void;                // 清除缓存渲染状态
}

render() 返回字符串数组（每行一个），每行可见宽度不得超出 width 参数，否则 TUI 会报错。TUI 在每行末尾自动追加 SGR reset 和 OSC 8 reset，因此样式不会跨行继承。

Focusable 接口与 IME 光标

Focusable 接口（packages/tui/src/tui.ts）支持 IME（输入法）候选窗口定位：

export interface Focusable {
  focused: boolean;  // TUI 在焦点变化时设置
}

// 零宽度 APC 序列，组件在光标位置输出
export const CURSOR_MARKER = "\x1b_pi:c\x07";

当 Focusable 组件获得焦点时，TUI 会： 1. 设置 focused = true 2. 扫描渲染输出中的 CURSOR_MARKER，计算其可视列位置 3. 将硬件终端光标定位到该位置（用于 IME 候选窗口定位）

通过 PI_HARDWARE_CURSOR=1 环境变量或 setShowHardwareCursor(true) 启用可见硬件光标。Editor 和 Input 组件已内置实现该接口。

差分渲染三策略

doRender()（packages/tui/src/tui.ts）根据场景选择三种渲染策略：

策略 触发条件 行为
首次渲染 previousLines.length === 0 且无宽高变化 直接输出所有行，不清除屏幕
宽度变化/变化超出视口 终端宽度改变 或 首个变化行在视口上方 或 内容收缩 全屏清除后完整重绘
正常更新 仅部分行内容变化 光标移动到首个变化行，清除到末尾，仅渲染变化行

所有渲染更新均包裹在 CSI 2026 同步输出（\x1b[?2026h ... \x1b[?2026l）中，保证原子性无闪烁渲染。渲染循环使用 16ms 最小间隔（MIN_RENDER_INTERVAL_MS）防止过度重绘。

Overlay 系统

TUI.showOverlay(component, options?)（packages/tui/src/tui.ts）在现有内容之上叠加模态组件，返回 OverlayHandle 控制生命周期：

  • 定位：支持 9 种锚点（center、top-left、bottom-right 等）、绝对坐标、百分比坐标和边距
  • 尺寸：width（绝对或百分比）、minWidth、maxHeight
  • 响应式可见性：visible(termWidth, termHeight) 回调按终端尺寸控制显隐
  • 焦点管理：overlayStack 按 focusOrder 排序，顶部可见 overlay 自动获得焦点；nonCapturing: true 可跳过焦点抢占
  • 焦点恢复：底层组件重新获得焦点时，overlay 的焦点恢复状态（overlayFocusRestore）决定是否自动重新聚焦 overlay

OverlayHandle 提供 hide()（永久移除）、setHidden()（临时隐藏）、focus()、unfocus() 等方法。Container 定义在 tui.ts 而非 components/ 目录，是组件树的根节点，递归渲染 children。

Terminal 接口与键盘协议

Terminal 接口

Terminal 接口（packages/tui/src/terminal.ts）抽象终端 I/O，TUI 通过该接口与任何终端实现交互：

export interface Terminal {
  start(onInput: (data: string) => void, onResize: () => void): void;
  stop(): void;
  write(data: string): void;
  get columns(): number;
  get rows(): number;
  moveBy(lines: number): void;
  hideCursor(): void;
  showCursor(): void;
  clearLine(): void;
  clearFromCursor(): void;
  clearScreen(): void;
}

ProcessTerminal（packages/tui/src/terminal.ts）是默认实现，使用 process.stdin/stdout。启动时执行以下初始化流程：

  1. 启用 raw mode（setRawMode(true)）
  2. 启用 bracketed paste mode（\x1b[?2004h）——终端将粘贴内容包裹在 \x1b[200~ ... \x1b[201~ 中
  3. 查询并启用 Kitty 键盘协议（渐进增强检测）
  4. 在 Windows 上启用 ENABLE_VIRTUAL_TERMINAL_INPUT（通过 native/win32/ 原生模块）
  5. 建立 StdinBuffer 进行输入缓冲与序列拆分

VirtualTerminal 基于 @xterm/headless，用于测试环境。

Kitty 键盘协议协商

ProcessTerminal 在启动时发送渐进增强查询（packages/tui/src/terminal.ts）：

\x1b[>7u    <- 请求 flags: 1(disambiguate) + 2(event types) + 4(alternate keys)
\x1b[?u     <- 查询当前 flags
\x1b[c      <- DA 查询（作为哨兵，不支持 Kitty 协议的终端会先响应 DA）

请求的 flags 含义： - flag 1：消除转义码歧义（所有按键都以 CSI u 格式发送） - flag 2：报告事件类型（press/repeat/release） - flag 4：报告 alternate keys（shift 后的字符 + 物理布局键码）

如果终端响应 Kitty flags，则激活 Kitty 协议。如果先收到 DA 响应，则回退到 modifyOtherKeys（\x1b[>4;2m）。停止时发送 \x1b[<u 关闭协议。

drainInput() 在退出前排空 stdin，防止 Kitty 按键释放事件通过慢速 SSH 连接泄漏到父 shell。

StdinBuffer

StdinBuffer（packages/tui/src/stdin-buffer.ts）将 stdin 数据拆分为完整序列。因为 stdin data 事件可能分片到达（尤其对于 escape 序列），缓冲累积不完整序列，通过超时（默认 10ms）机制刷新：

  • 识别完整的 CSI、OSC、DCS、APC、SS3 序列
  • 特殊处理 bracketed paste（\x1b[200~ ... \x1b[201~），paste 内容作为 paste 事件发出
  • 处理 WezTerm 下 Escape 键 press/release 的拼接问题

键盘输入系统

keys.ts：matchesKey 与 Kitty 协议解析

keys.ts（packages/tui/src/keys.ts）是键盘输入处理的核心，支持三种协议路径：

  1. Kitty 键盘协议（CSI u 格式）：\x1b[codepoint;modifier:eventu
  2. modifyOtherKeys（xterm 格式）：\x1b[27;modifier;codepoint~
  3. Legacy 序列：传统 ANSI escape 序列（\x1b[A = up 等）

matchesKey(data, keyId) 按优先级依次匹配三种协议，对字母键进行 normalizeShiftedLetterIdentityCodepoint 归一化（shift+字母 → 小写 codepoint），并对非拉丁键盘布局通过 baseLayoutKey 回退匹配。parseKey(data) 反向解析原始输入为 key identifier 字符串。

Key 助手对象提供类型安全的键标识符创建：

Key.escape          // "escape"
Key.ctrl("c")       // "ctrl+c"
Key.ctrlShift("p")  // "ctrl+shift+p"
Key.super("k")      // "super+k"

isKeyRelease(data) 通过检测 Kitty 协议 event type 标记（:3 后缀）识别按键释放事件。decodeKittyPrintable(data) 从 Kitty CSI-u 序列中提取可打印字符。

keybindings.ts：可配置按键绑定

KeybindingsManager（packages/tui/src/keybindings.ts）提供全局按键绑定注册表，支持用户自定义覆盖默认按键。默认绑定（TUI_KEYBINDINGS）涵盖：

  • Editor 导航/编辑：tui.editor.cursorUp/Down/Left/Right、tui.editor.deleteWordBackward（ctrl+w/alt+backspace）、tui.editor.jumpForward（ctrl+]）、tui.editor.jumpBackward（ctrl+alt+]）、tui.editor.undo（ctrl+-）、tui.editor.yank（ctrl+y）/yankPop（alt+y）等
  • 通用输入：tui.input.newLine（shift+enter/ctrl+j）、tui.input.submit（enter）、tui.input.tab（tab）
  • 选择操作：tui.select.up/down、tui.select.confirm（enter）、tui.select.cancel（escape/ctrl+c）

通过声明合并（declaration merging），下游包可扩展 Keybindings 接口添加自定义绑定。冲突检测会报告同一按键映射到多个绑定的情况。

组件系统

pi-tui 提供 13 个内置组件：12 个在 packages/tui/src/components/ 目录，Container 在 tui.ts 中定义。

Text 与 TruncatedText

Text（packages/tui/src/components/text.ts）显示多行文本，支持自动换行、内边距和自定义背景色函数。通过 setText() 动态更新内容。

TruncatedText（packages/tui/src/components/truncated-text.ts）单行文本，超出视口宽度时截断并追加省略号。适用于状态栏和标题行。

Input

Input（packages/tui/src/components/input.ts）单行文本输入，支持水平滚动、onSubmit 回调。内置 Emacs 风格键绑定（ctrl+a/e 行首/尾、ctrl+w 删除单词、ctrl+u/k 删除到行首/尾）和词级导航（ctrl+left/right、alt+left/right）。

Editor

Editor（packages/tui/src/components/editor.ts）是多行文本编辑器的核心实现。主要特性：

  • 多行编辑与自动换行：基于 Unicode 字素分割（Intl.Segmenter）的精确换行，通过 segmentWithMarkers() 将 paste marker 作为原子字素处理
  • 自动补全：集成 CombinedAutocompleteProvider，支持斜杠命令和文件路径补全
  • Paste marker：超过 10 行的粘贴内容被折叠为 [paste #N +M lines] 标记，作为不可部分操作的原子单位
  • Kill ring（packages/tui/src/kill-ring.ts）：Emacs 风格 kill/yank，连续删除累积到同一条目，yankPop（alt+y）循环历史
  • Undo/Redo：基于 UndoStack（packages/tui/src/undo-stack.ts），每次编辑操作前 push structuredClone 快照，ctrl+- 触发 undo
  • Jump-to-char：ctrl+] 向前跳转到指定字符，ctrl+alt+] 向后跳转
  • 主题支持：EditorTheme 接口控制边框颜色和下拉列表样式

Markdown

Markdown（packages/tui/src/components/markdown.ts）将 Markdown 文本渲染为 ANSI 样式行。支持标题、粗体、斜体、代码块（可选语法高亮 highlightCode）、列表、链接、引用块、分割线等。HTML 标签以纯文本形式渲染。内部缓存渲染结果，invalidate() 清除缓存。

Loader 与 CancellableLoader

Loader（packages/tui/src/components/loader.ts）动画加载旋转器，通过定时器触发 tui.requestRender() 实现动画帧更新。setMessage() 动态更新提示文本。

CancellableLoader（packages/tui/src/components/cancellable-loader.ts）扩展 Loader，添加 Escape 键监听和 AbortSignal 支持。signal 属性在用户按 Escape 时被 abort，onAbort 回调通知调用方取消异步操作。

SelectList 与 SettingsList

SelectList（packages/tui/src/components/select-list.ts）交互式选择列表。支持键盘导航（方向键）、Enter 确认、Escape 取消、onSelectionChange 高亮回调。maxVisible 控制可见项数，超出时显示滚动信息。

SettingsList（packages/tui/src/components/settings-list.ts）设置面板，支持值循环（values 数组）和子菜单（submenu 函数返回 Component）。方向键导航，Enter/Space 激活当前项。

Box

Box（packages/tui/src/components/box.ts）带内边距和背景色的容器。构造参数 paddingX、paddingY 和 bgFn 函数（为每一行应用背景色）。setBgFn() 动态更换背景色。

Image

Image（packages/tui/src/components/image.ts）终端内联图像渲染。支持 Kitty（\x1b_G...）和 iTerm2（\x1b]1337;File=...）图形协议，不支持的终端回退到文本占位符。支持 PNG、JPEG、GIF、WebP 格式，从二进制头部自动解析尺寸。通过 maxWidthCells/maxHeightCells 限制渲染尺寸。

Spacer

Spacer（packages/tui/src/components/spacer.ts）垂直空白行，构造参数为行数（默认 1）。

Container

Container（packages/tui/src/tui.ts）组件树的根节点和分组容器。children 数组按序递归渲染，提供 addChild()、removeChild()、clear() 方法。TUI 继承自 Container。

自动补全系统

CombinedAutocompleteProvider（packages/tui/src/autocomplete.ts）统一处理斜杠命令和文件路径补全：

  • 斜杠命令：输入 / 触发，从命令列表模糊匹配（使用 fuzzyFilter）
  • 命令参数补全：命令对象的 getArgumentCompletions(argumentPrefix) 方法提供上下文补全
  • 文件路径：Tab 键触发，使用 readdirSync 同步枚举目录
  • @ 前缀模糊搜索：使用外部 fd 工具递归搜索文件树（尊重 .gitignore），适合大项目快速文件定位
  • 引号处理：自动处理 "path with spaces" 和 @"file name" 格式

终端图像渲染

terminal-image.ts（packages/tui/src/terminal-image.ts）实现终端内联图像渲染：

  • 能力检测：通过 TERM_PROGRAM、KITTY_WINDOW_ID、ITERM_SESSION_ID 等环境变量识别终端类型，支持 Kitty、Ghostty、WezTerm、Warp（Kitty 协议）和 iTerm2
  • Kitty 协议：encodeKitty() 生成 \x1b_Ga=T,f=100,q=2,c=...;base64\x1b\\ 格式，分块传输（4096 字节/块）
  • iTerm2 协议：encodeITerm2() 生成 \x1b]1337;File=inline=1;...;base64\x07 格式
  • 尺寸计算：calculateImageCellSize() 基于终端 cell 像素尺寸（通过 CSI 16 t 查询）计算目标行列数
  • 图像管理：allocateImageId() 分配随机 ID，deleteKittyImage() 和 deleteAllKittyImages() 管理 Kitty 图像生命周期
  • 超链接：hyperlink() 生成 OSC 8 格式的可点击超链接

工具函数

utils.ts

utils.ts（packages/tui/src/utils.ts）提供核心字符串处理函数：

  • visibleWidth(str)：计算字符串在终端中的可见宽度，基于 Intl.Segmenter 字素分割和 get-east-asian-width 库，正确处理全角字符、emoji、ANSI 转义码
  • truncateToWidth(text, maxWidth, ellipsis?, pad?)：按可见宽度截断字符串，保留 ANSI 码完整性
  • wrapTextWithAnsi(text, width)：自动换行，保留 ANSI 样式跨行，正确处理 OSC 8 超链接
  • extractSegments()：单次遍历提取行中 "before" 和 "after" 片段，用于 overlay 合成
  • sliceByColumn()：按列位置切片，处理宽字符边界
  • normalizeTerminalOutput()：归一化泰文/老挝文 AM 元音的组合形式

fuzzy.ts

fuzzy.ts（packages/tui/src/fuzzy.ts）实现模糊匹配算法，支持多 token（空格/斜杠分隔）、字母-数字互换匹配、词边界加分、连续匹配加分。

stdin-buffer.ts

StdinBuffer（packages/tui/src/stdin-buffer.ts）缓冲 stdin 输入并拆分为完整序列。基于 OpenTUI 实现（MIT 协议），处理 CSI、OSC、DCS、APC 序列的完整性判断，支持 bracketed paste 模式。

kill-ring.ts

KillRing（packages/tui/src/kill-ring.ts）Emacs 风格 kill ring。push(text, opts) 支持累积模式（连续删除合并），rotate() 实现 yank-pop 循环。

undo-stack.ts

UndoStack<S>（packages/tui/src/undo-stack.ts）通用撤销栈，push 时自动 structuredClone 深拷贝状态快照。

word-navigation.ts

word-navigation.ts（packages/tui/src/word-navigation.ts）提供 findWordBackward() 和 findWordForward()，基于 Intl.Segmenter 词分割实现词级光标移动，支持原子段（paste marker）和标点边界。

native/ 平台原生模块

packages/tui/native/ 包含两个平台的原生 Node.js addon：

  • native/darwin/（darwin-modifiers.node）：macOS 修饰键实时检测。isNativeModifierPressed()（packages/tui/src/native-modifiers.ts）查询 shift/command/control/option 键的物理按下状态。用于 Apple Terminal 下区分 Shift+Enter（原生 \r 无法区分）
  • native/win32/（win32-console-mode.node）：Windows 控制台模式设置。enableVirtualTerminalInput() 向 stdin 句柄添加 ENABLE_VIRTUAL_TERMINAL_INPUT（0x0200）标志，使 Windows Terminal 发送 VT 转义序列而非原始控制台事件（否则 Shift+Tab 会丢失修饰键信息变为普通 \t）

与上层的关系

pi-tui 被 pi-coding-agent 直接依赖（packages/coding-agent/package.json）。上层通过 TUI + ProcessTerminal 组合创建交互式终端界面，使用 Editor 作为主输入组件，Markdown 渲染 AI 响应，Loader/CancellableLoader 显示加载状态，SelectList 实现选项菜单，showOverlay() 实现弹窗和对话框。
