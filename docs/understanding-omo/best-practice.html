最佳实践 & 快速参考

刚装完 OmO 不知道从哪开始？这份指南带你走完第一个任务，然后告诉你每种场景用什么工具最顺手。

首次使用

Step 1: 安装

bunx oh-my-openagent install

安装向导会检测 OpenCode 版本、配置权限、复制 agent 定义和 skill 文件。完成后重启 OpenCode。

没有推荐的模型订阅？没关系。
如果安装时跳过了模型配置，或者你只有部分 provider（比如只有 OpenCode Go 或某个免费 API），直接在 OpenCode 里告诉 Sisyphus：

我没有 Claude 和 GPT 的订阅。帮我检查 opencode models，然后用我可用的模型重新配置所有 agent。

Sisyphus 会自动运行 opencode models 查看你的可用模型，然后按 Agent 的回落链（fallback chain）给每个 Agent 分配合适的模型——不需要手动编辑 JSON 配置。

Step 2: 运行第一个任务

启动 OpenCode，在输入框里直接打：

ultrawork 帮我给项目的 README.md 加一个安装说明章节

对，就这一句话。OmO 的 IntentGate 会自动识别你的意图是 ultrawork 模式，激活 Sisyphus 编排器做规划、委派和执行。不需要选 agent、不需要配模型、不需要写提示词。

Step 3: 了解 Agent 面板

OpenCode 的 agent 选择面板（通常在输入框上方）会列出所有可用的 Agent：

Agent按什么键做什么
Sisyphus默认 / Tab主编排器。自动规划、委派、执行。80% 的场景用它。
HephaestusTab 切换自主深度工作。给目标不给配方，自己搞。
PrometheusTab 切换战略规划。像工程师一样访谈你，然后出计划。
AtlasTab 切换TODO 编排器。按计划逐项执行，/start-work 激活。
OracleTab 切换只读顾问。不改代码，只给建议。
ExploreTab 切换快速代码库搜索。grep 一把梭。
LibrarianTab 切换文档 / 开源代码研究。深度搜索。
Multimodal LookerTab 切换看图片、PDF、图表的。
MetisTab 切换规划顾问。在计划定稿前找漏洞。
MomusTab 切换计划审查。验证可执行性。
Sisyphus-Junior自动触发Category 衍生执行器。由 Sisyphus 自动委派。

提示：按 Tab 键可以快速在 agent 之间切换。大部分时间你只需要 Sisyphus，其他 agent 会在需要时被自动调用。

Step 4: 第一个真实任务

假设你正在开发一个 React 项目，想加一个暗色模式切换：

  1. 在 OpenCode 输入 ulw add dark mode toggle to my React app，回车
  2. Sisyphus 分析请求，激活 ultrawork 模式
  3. 它会自动：读取项目结构 → 判断现有样式方案 → 创建 Context + Hook + Toggle 组件 → 修改入口文件 → 验证不破坏已有 UI
  4. 过程中你不需要做任何事。如果 agent 问问题，回答就行
  5. 完成后 agent 会展示变更摘要。用 /diff 审查变更，满意了让它提交

整个过程可能是 2–5 分钟，取决于项目复杂度。你只需要说一句话。

什么情况用哪个 Agent

不是所有任务都需要思考"用哪个 agent"—Sisyphus 默认会帮你做选择。但了解每个 agent 的强项能让你在复杂场景下更精准地委派。

场景推荐 Agent操作方式原因
日常开发任务 Sisyphus 默认，直接对话 主编排器，自动规划和委派
复杂但懒得解释 Sisyphus 输入 ulw 或 ultrawork 激活 ultrawork 模式，agent 自行探索
复杂且需要精确规划 Prometheus → Atlas @plan "任务描述" → 审查 → /start-work Prometheus 访谈规划，Atlas 逐步执行
深度架构推理 Hephaestus Tab 切换到 Hephaestus GPT-5.5 自主深度推理
需要架构建议（不改代码） Oracle @oracle 你的问题 只读高智商顾问
查找代码 / 文档 Explore / Librarian @explore 找... / @librarian 查... 快速 grep / 文档搜索
多成员并行协作 Team Mode 启用 team_mode，用 team_create 并行多 agent 系统
快速修复（typo / 单文件） Sisyphus 直接说明 无需规划，直接执行
安全审计 Sisyphus + skill ulw security-research 3 个漏洞猎人 + 2 个 PoC 工程师并行

决策流程

不知道怎么选？按这个流程走：

1
任务简单吗？（改一个 typo、调个样式、加个注释）
→ 是：直接跟 Sisyphus 说，不需要任何特殊操作
→ 否：往下走
2
懒得写详细需求？
→ 是：输入 ulw 或 ultrawork 开头，只说一句话。Agent 自己探索代码库、自己规划、自己实现。
→ 否：往下走
3
需要精确、可验证的方案？
→ 是：@plan "任务描述" → Prometheus 会像真正的工程师一样提问访谈，生成详细计划 → 你审查计划 → /start-work 开始执行。
→ 否：往下走
4
需要 GPT 风格的深度推理？
→ 是：按 Tab 切换到 Hephaestus。给他目标，不给他步骤。
→ 否：往下走
5
只是问问题，不改代码？
→ 是：@oracle 你的问题。Oracle 只读不改，纯粹当顾问。
→ 否：往下走
6
需要多人并行协作？
→ 配置 team_mode.enabled: true → 重启 → team_create 定义团队。
→ 以上都不适用？直接用 Sisyphus 说人话就对了。

实用技巧

记住一个原则：你只管说目标，Sisyphus 搞定一切。
不需要写 task()、不需要选模型、不需要配参数。
你输入的是自然语言——"修这个 bug"、"加个暗色模式"、"审查这段代码"。
Sisyphus 自动判断复杂度、选 Category、加载 Skill、委派子 agent。
下面列出的 task()、load_skills 等是 Sisyphus 的内部机制——了解即可，你不需要手写。

TODO 强制执行

Agent 偷懒或跑偏时，系统自动推送 <system-reminder> 把它拉回来。不需要手动催促。

会话恢复

会话断了？打开新会话，输入 /start-work 就能恢复。进度、上下文、todo 都不会丢。

后台并行

Sisyphus 会自动在后台并行启动多个子 agent 同时工作——搜索代码、查文档、写代码同步进行。你不需要手动操作。

内部机制：Sisyphus 通过 task(run_in_background=true) 并行委派，上限按 provider/model 组合独立计算。

Skill — 给 Agent 装专业知识

Skill 是什么？ 一套预设的领域知识 + 工具 + 工作流。加载后 Agent 从通用型变专家。

你怎么用？ 在对话框里直接说：

用 frontend skill 帮我设计一个响应式导航栏

或者更直接：加载 playwright skill，帮我写 e2e 测试

Skill 存在哪里？ 项目 .opencode/skills/<name>/SKILL.md、用户 ~/.config/opencode/skills/、以及内置（frontend-ui-ux、git-master、playwright 等）。

内部机制：Sisyphus 通过 task(category="...", load_skills=["..."]) 把 skill 注入子 agent 的 prompt——你不需要手动写这个。

Category 路由 — 无需指定模型

你不需要告诉 Sisyphus 用哪个模型。它根据任务类型自动选择。

你怎么说：

帮我重构这个认证模块（复杂逻辑 → 自动走 ultrabrain）
帮我修一下这个 typo（简单任务 → 自动走 quick）
帮我还原刚才的提交（git 操作 → 自动加载 git-master skill）

内部机制：Sisyphus 通过 task(category="ultrabrain") 把任务路由到最优模型。Category + Skill 自动组合——你只说目标，系统搞定技术细节。

LSP 诊断

每次代码变更后自动跑 lsp_diagnostics。agent 不会交烂代码——LSP 检查不过它不会停。

Hashline 编辑

每行代码都有 LINE#ID 内容哈希保护。编辑时如果行内容变了，agent 会自动重读文件而不是写烂代码。

健康检查

出问题了？

bunx oh-my-opencode doctor

检查系统、配置、工具、模型四个维度。
